# Доступ к AWS с помощью статических ключей (не рекомендуется)

<p style="background-color: #ffdddd; border-left: 6px solid #f44336; padding: 15px;">
  <strong>ВНИМАНИЕ:</strong> Этот метод использует долгоживущие ключи доступа и считается менее безопасным. Используйте его только в крайних случаях. Рекомендуемый способ — <a href="/ru/secure-access/aws-oidc.md">настройка доступа через OIDC</a>.
</p>

Ключи доступа AWS (Access Key ID и Secret Access Key) необходимы для программного взаимодействия с вашим аккаунтом AWS, например, для использования Terraform или AWS CLI.

## Пошаговая инструкция

1.  **Войдите в AWS Management Console:**
    Откройте [консоль AWS](https://aws.amazon.com/console/) и войдите в свой аккаунт.

2.  **Перейдите в сервис IAM:**
    В строке поиска вверху введите `IAM` и выберите "IAM" из списка сервисов.

3.  **Перейдите к пользователям:**
    В левой навигационной панели выберите **Users**.

4.  **Выберите или создайте пользователя:**
    *   Создайте нового IAM-пользователя. **Не используйте ключи корневого пользователя (root user).**
    *   Нажмите на имя существующего пользователя или нажмите **"Create user"**.

5.  **Перейдите на вкладку "Security credentials":**
    Внутри страницы управления пользователем откройте вкладку **Security credentials**.

6.  **Создайте ключ доступа:**
    *   Прокрутите вниз до раздела **"Access keys"**.
    *   Нажмите кнопку **"Create access key"**.

7.  **Выберите вариант использования:**
    *   Выберите **"Command Line Interface (CLI)"**.
    *   Нажмите **"Next"**.

8.  **Сохраните ваши ключи:**
    *   На последнем шаге AWS покажет вам **Access key ID** и **Secret access key**.
    *   **Это единственный раз, когда вы сможете увидеть Secret Access Key.** Скопируйте и сохраните оба ключа в надежном месте.

9.  **Используйте ключи в GitHub Secrets:**
    *   Перейдите в настройки вашего репозитория на GitHub.
    *   Откройте **Settings** -> **Secrets and variables** -> **Actions**.
    *   Создайте два секрета:
        *   `AWS_ACCESS_KEY_ID`: Вставьте сюда ваш Access Key ID.
        *   `AWS_SECRET_ACCESS_KEY`: Вставьте сюда ваш Secret Access Key.
