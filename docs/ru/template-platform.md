# Template Platform

**Template Platform** — это фундаментальный бэкенд-сервис, спроектированный как готовая к использованию, масштабируемая и поддерживаемая отправная точка для новых приложений. Он без проблем интегрируется с инфраструктурными компонентами из `infra-core` и `infra-template`, а его развертывание управляется пайплайнами из `infra-ci`.

## Основное назначение

Главная цель `template-platform` — ускорить разработку, предоставляя предварительно настроенное окружение, которое включает:

- **Каркас API**: Базовая структура для создания RESTful или GraphQL API.
- **Интеграция с базой данных**: Преднастроенные подключения к стандартным базам данных.
- **Интеграция с CI/CD**: Готовность к развертыванию в `dev` и `stage` окружения "из коробки" с использованием `infra-ci`.
- **Наблюдаемость (Observability)**: Хуки для логирования, метрик и трассировки.

## Начало работы

Чтобы начать работу с `template-platform`, убедитесь, что вы клонировали репозиторий и инициализировали его подмодули:

```bash
git clone <your-repo-url>/template-platform.git
cd template-platform
git submodule update --init --recursive
```

Сервис можно запустить локально с помощью Docker Compose:

```bash
docker-compose up --build
```

## Развертывание

Развертывание выполняется автоматически через GitHub Actions при пуше в ветку `main` (для `dev` окружения) или может быть запущено вручную (для `stage` окружения). Пайплайн использует конфигурации Terraform, определенные в репозитории `template-platform`, которые, в свою очередь, используют модули из `infra-template`.
