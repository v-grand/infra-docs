# Статические ключи AWS

!!! danger "Не рекомендуется"
    Использование статических ключей доступа IAM **не рекомендуется** для программного доступа. По возможности всегда предпочитайте временные учетные данные, используя такие методы, как **[AWS OIDC](aws-oidc.md)**. Статические ключи — это долгоживущие учетные данные, которые могут представлять значительный риск безопасности в случае компрометации.

## Обзор

Этот метод включает создание пользователя IAM с выделенным набором долгосрочных ключей доступа (`Access Key ID` и `Secret Access Key`). Эти ключи затем настраиваются как переменные окружения или в файле учетных данных, которые AWS SDK или CLI используют для аутентификации запросов.

## Случаи использования

Используйте статические ключи только тогда, когда временные учетные данные не являются жизнеспособным вариантом, например:
- Устаревшее приложение, которое не поддерживает современные потоки аутентификации.
- Специфические сторонние инструменты, требующие долгоживущих ключей.
- Сценарии начальной настройки или "bootstrapping" до того, как могут быть настроены более безопасные методы.

## Конфигурация

Если вам необходимо использовать статические ключи, настройте их как переменные окружения:

```bash
export AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
export AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
export AWS_REGION=us-east-1
```

## Лучшие практики безопасности

- **Принцип наименьших привилегий**: Предоставляйте пользователю IAM только те разрешения, которые ему абсолютно необходимы.
- **Ротация ключей**: Регулярно меняйте ключи доступа, чтобы ограничить окно возможностей для злоупотребления в случае компрометации ключа.
- **Мониторинг**: Используйте AWS CloudTrail для мониторинга всех вызовов API, сделанных с помощью статических ключей. Настройте оповещения о необычной активности.
- **Не храните ключи в коде**: Никогда не встраивайте ключи доступа непосредственно в ваш код. Используйте переменные окружения или безопасный инструмент для управления секретами.
